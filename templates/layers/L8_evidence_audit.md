# L8 — Evidence Audit

## Purpose

Audit execution outputs for traceability, quality, and supportable findings before interpretation.

## Reasoning boundary

- Separate observed data, derived results, quality limitations, and unsupported claims.
- Trace each key result to an execution artifact and its provenance.
- Escalate missing, inconsistent, or low-quality evidence instead of smoothing it over.

## Handoff

Provide an evidence audit and bounded findings for literature verification and later review. Do not make final scientific conclusions or decisions.

## Full-mode role

The current dynamic contract is authoritative for inputs, output schema, state changes, and runtime commands.
