# L6 — Analysis Plan Decision

## Purpose

Decide whether a concrete analysis plan is ready to authorize, revise, or reject after method critique.

## Reasoning boundary

- Require prespecified inputs, procedures, quality checks, and decision criteria.
- Evaluate executability without treating it as proof of the hypothesis.
- Preserve the distinction between an approved plan and an achieved result.

## Handoff

Issue only the plan-level decision and its audit trail. Do not execute the plan, fabricate outputs, or select among scientific hypotheses.

## Full-mode role

The current dynamic contract is authoritative for inputs, output schema, state changes, and runtime commands.
